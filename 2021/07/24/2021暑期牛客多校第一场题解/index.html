A Alice and Bob

题意：

有两堆石子，一堆有n个石子，一堆有m个石子，每次可以从其中一堆石子中取k个（k > 0），然后从另一堆石子取s * k个（s >= 0），谁没法操作谁输，A先手是否能够必胜？

1 <= t <= 1e4, 1 <= n, m <= 5e3

题解：

先依次列举所有的必败态，容易找到前几个必败态：(0, 0)，(2, 3)，(5, 7)，这时候可以根据这些必败态推出所有必胜态。

f(i, j) –> f(i + k, j + s * k) or f(i + s * k, j + k)

ps：本题记忆化搜索会tle，直接从小到大正向递推即可，同时f数组int类型也会tle，改为bool会很快

代码：

#include<bits/stdc++.h>
using namespace std;
const int maxn = 5e3 + 10;
int T, n, m;
bool f[maxn][maxn];
 
void init()
{
    for(int i = 0; i <= 5000; i ++)
        for(int j = 0; j <= 5000; j ++)
            if(f[i][j] == 0)
            {
                for(int k = 1; i + k <= 5000; k ++)
                    for(int s = 0; j + s * k <= 5000; s ++)
                        f[i + k][j + s * k] = 1;
                for(int k = 1; j + k <= 5000; k ++)
                    for(int s = 0; i + s * k <= 5000; s ++)
                        f[i + s * k][j + k] = 1;
            }
}
 
void solve()
{
    scanf("%d%d", &n, &m);
    if(f[n][m]) puts("Alice");
    else puts("Bob");
}
 
int main()
{
    init();
    scanf("%d", &T);
    while(T --)
        solve();
    return 0;
}

D Determine the Photo Position

题意：

题解：

签到

代码：

#include<bits/stdc++.h>
using namespace std;
int n, m, mat[2010][2010];

int main()
{
    scanf("%d%d", &n, &m);
    for(int i = 1; i <= n; i ++)
        for(int j = 1; j <= n; j ++)
            scanf("%1d", &mat[i][j]);
    
    int res = 0;
    for(int i = 1; i <= n; i ++) {
        int cnt = 1;
        for(int j = 2; j <= n + 1; j ++) {
            if(j == n + 1 || mat[i][j] != mat[i][j - 1])
            {
                if(mat[i][j - 1] == 0 && cnt >= m) res += cnt - m + 1;
                cnt = 1;
            }
            else ++ cnt;
        }
    }
    
    printf("%d\n", res);
    
    return 0;
}

F Find 3-friendly Integers

题意：

每次给一个区间[l, r]，求在这个区间中有几个数满足是“3好数”，“3好数”的定义为：数字中有某一个子串为3的倍数，如104中 0 mod 3 = 0，124中 12 mod 3 = 0，17则不是。

题解：

规律观察题，明显可以发现从100到999所有数都不满足定义，则退出大于等于100的数都不满足，那么暴力枚举100以内的数即可。

代码：

#include<bits/stdc++.h>
using namespace std;
typedef long long ll;
int T, s[110];
ll l, r;

void init()
{
    for(int i = 1; i < 10; i ++) s[i] = s[i - 1] + (i % 3 == 0);
    for(int i = 10; i < 100; i ++) {
        int a = i / 10, b = i % 10;
        if(a % 3 == 0 || b % 3 == 0 || i % 3 == 0) s[i] = s[i - 1] + 1;
        else s[i] = s[i - 1];
    }
}

int main()
{
    init();
    scanf("%d", &T);
    while(T --)
    {
        scanf("%lld%lld", &l, &r);
        if(l >= 100) printf("%lld\n", r - l + 1);
        else if(r < 100) printf("%d\n", s[r] - s[l - 1]);
        else printf("%lld\n", 0ll + s[99] - s[l - 1] + r - 99);
    }

    return 0;
}

G Game of Swapping Numbers

题意：

给定两个长度为n的序列a，b，每次操作可以交换a序列中两个不同位置的数，求在恰好k次操作下，使$ \sum^{n}_{i=1}{|a_i-b_i|} $最大，求出最大值。

2 <= n <= 5e5, 0 <= k <= 1e8, -1e8 <= ai, bi <= 1e8

题解：

贪心结论题。由于本人太菜，只好对题解进行复述。

对于该题，看完题目最先想到的应该是最优解满足哪些性质，emmm，似乎也没有什么更进一步的思路。

对于求两个值差的绝对值，比较容易想到的一点就是，把每一对a和b当成一个区间，本题的答案相当于在数轴上求所有区间的总和，把每一个ai，bi看成一个个区间。那么这时候可以思考一下如何交换a中的数，会对数轴上的区间有何影响。

容易发现，对于两个区间(a1, b1)，(a2, b2)，操作它们对其他区间毫无影响，那么先考虑这两个区间。

根据这两个区间是否相交可以分成2种情况，而根据a1，a2的位置，又对应能够分成4种情况：

首先先看相交的4种情况：

情况1：

交换之后变成：

发现对答案的贡献没有变化

情况2：

交换之后变成：

发现贡献变少了。

相对应了情况3与情况4，就是分别枚举了一下a1, a2的位置，可以自行画图。最后可以发现，当a1，a2同侧的时候交换是不会改变贡献的，异侧的时候会减少贡献。

总而言之，如果两个区间相交了，那么交换对应的a值不会使得解更优。

对应的，当区间不相交的时候，再次分别枚举a1，a2所在位置，又对应4种情况，容易发现，这4种情况交换总能够使贡献变大（每次合并区间所带来的贡献为它们中间相隔的距离的2倍）。

同时可以发现，把a序列，b序列像这样全部都抽象成一个个区间，那么左端点是a是b就不重要了，因为我们的任务从交换a中的数变成了合并区间。这时候我们可以假设ai<=bi，如果不满足交换一下即可。（这里可以结合上面的区间合并想一想，交换a，b是不影响最终解的，因为我们的任务已经变成了合并区间，而不是交换a中的数）。

那么根据贪心，只需要优先把最左边的区间和最右边的区间合并即可。

那么接下来还有一个问题：就是如何做到恰好k次？

可以证明：当n>2时，在最优解中一定存在一种交换方式，交换之后还是最优解。

这个依然用区间合并来解释。首先，k次操作以内一定达到了最优解，那么所有区间一定都是相交的。这时候仔细回头看看相交区间的那4种情况，a1，a2必然是同侧的（因为已经交换过了让ai<=bi），所以不管怎么交换都是答案都是不变的。

最后关于代码的一点解释：

根据贪心来进行每一次合并，那么我们就应该让最小的b和最大的a合并（同时bi < ai），多产生的贡献为(ai - bi) * 2，此时的a和b绝对是两个不同的区间，因为ai < bi，如果a > b了那么合并的这两个点绝对不是同一个区间，一直循环到bi>=ai的时候退出，因为后面都不会产生任何贡献了。

所以a，b对应交换完了之后，排序并判断即可。

代码：

#include<bits/stdc++.h>
using namespace std;
const int maxn = 5e5 + 10;
int n, a[maxn], b[maxn], k;
long long ans = 0;

int main()
{
    scanf("%d%d", &n, &k);
    for(int i = 1; i <= n; i ++) scanf("%d", a + i);
    for(int i = 1; i <= n; i ++) scanf("%d", b + i);
    
    if(n == 2) {
        printf("%d", (k & 1) ? abs(a[1] - b[2]) + abs(a[2] - b[1]) : abs(a[1] - b[1]) + abs(a[2] - b[2]));
        return 0;
    }
    
    for(int i = 1; i <= n; i ++) if(a[i] > b[i]) swap(a[i], b[i]);
    for(int i = 1; i <= n; i ++) ans += b[i] - a[i];
    sort(a + 1, a + n + 1); reverse(a + 1, a + n + 1);
    sort(b + 1, b + n + 1);

    for(int i = 1; i <= k; i ++) {
        if(a[i] - b[i] > 0) ans += (a[i] - b[i]) << 1;
        else break;
    }

    printf("%lld\n", ans);

    return 0;
}
