基本概念与定理

流网络

（先不考虑反向边）

1是源点，4是汇点，流量从源点流出，一直流到汇点，边上的权值是该边的容量。

上图称为一个流网络

可行流

（不考虑反向边）

规定f(u,v)为从u流向v的流向。

图中绿色边为一个从源点流向汇点的流量，这个为一个可行流。

可行流满足以下两个条件：

1.容量限制：某条边上的流量不能大于它的容量

$$ 0 \leq f(u, v) \leq c(u, v) $$

2.流量守恒：对某个除了源点和汇点的点来说，流进该点的流量等于流出该点的流量

$$ \forall x \in V / \begin{Bmatrix} s,t \end{Bmatrix}, \sum_{(u, x) \in E}{f(u, x)} = \sum_{(x, v) \in E}{f(x, v)} $$

可行流的流量指的是从源点流出的流量-流入源点的流量，即$ f(s, V’)-f(V’’, s) $

最大流指的是最大可行流

残留网络

一个可行流通过绿色边从源点流向汇点之后，对原图的每一条边建立一条反向边，其容量设为c(v, u) = c(u, v) - f(u, v)。

注意：一个残留网络$ G_f $是相对于可行流$ f $来说的

对于残留网络来说，残留网络的可行流f’ + 原图的可行流f = 另外一个可行流

证明：首先，残留网络中的边分两种，一种是原图的正向边，容量为c(u, v) - f(u, v)，一种是建立的反向边，容量为f(u, v)。

要证明新的流为可行流那么只需要证明其满足两条性质即可。

$$ \begin{aligned} 0 \leq f(u, v) \leq c(u, v) \hfill \\ 对于正向边 0 \leq f’(u, v) \leq c(u, v)-f(u, v) \Rightarrow 0 \leq f’(u, v) + f(u, v) \leq c(u, v) \\ 对于反向边 f’(u, v) \leq f(u, v) \Rightarrow 0 \leq f(u, v) - f’(u, v) \leq c(u, v) \\ 可见满足性质一 \\ \end{aligned} $$

$$ 对于某一个点，原图中流入该点的流量设为x_1，流出该点的流量设为y_1 \\ 残留网络中流入该点的流量设为x_2，x_3（分别对应原边和反向边），残留网络中流入该点的流量设为y_2，y_3（分别对应原边和反向边）\\ 把流入的反向边看成后悔流出的量，流出的反向边看成后悔流入的量 \\ 那么新的可行流流入量为x_1 + x_2 - y_3，流出的量y_1 + y_2 - x_3，由x_1=y_1，x_2+x_3=y_2+y_3可知前两式相等 \\ 满足性质二 $$

增广路径

在一个残留网络中，从源点沿着大于0的边一直走到汇点的一条路径。

割

对于一张流网络G(V, E)，考虑两个点集S，T，满足$ s \in S, t \in T, S \cup T = V, S \cap T = \phi $，这样把一张流网络分成两部分的一张图，称为一个割。

割的容量：不考虑反向边，所有S中某个点连着T中某个点的边，其容量之和为割的容量。

$$ c(S, T) = \sum_{u \in S} \sum_{v \in T}{c(u, v)} $$

“最小割”指的是容量最小的割

割的流量：考虑反向边，所有S中某个点连着T中某个点的边，其流量之和为割的流量，f(S, T) <= c(S, T)

$$ f(S, T) = \sum_{u \in S} \sum_{v \in T}{f(u, v)} - \sum_{u \in T} \sum_{v \in S}{f(u, v)} $$

关于割的流量有如下几个性质：

$$ f(X, Y) = -f(Y, X) \\ f(X, X) = 0 \\ X \cap Y = \phi, f(Z, X \cup Y) = f(Z, X) + f(Z, Y) \\ X \cap Y = \phi, f(X \cup Y, Z) = f(X, Z) + f(Y, Z) $$

割的相关定理

定理一：对于任意的可行流f，任意割[S, T]，|f| = f(S, T)

证明：

$$ f(S, V) = f(S, S) + f(S, T) \Rightarrow \\ f(S, T) = f(S, V) - f(S, S) = f(S, V) = f(\begin{Bmatrix} s \end{Bmatrix}, V) + f(S - \begin{Bmatrix} s \end{Bmatrix}, V) \\ 考查 f(S - \begin{Bmatrix} s \end{Bmatrix}, V), 令S’ = S - \begin{Bmatrix} s \end{Bmatrix} \\ f(S’, V) = \sum_{u \in S’} \sum_{v_1 \in V}{f(u, v_1)} - \sum_{u \in S’} \sum_{v_2 \in V}{f(v_2, u)} = \sum_{u \in S’}(\sum_{v_1 \in V}{f(u, v_1)} - \sum_{v_2 \in V}{f(v_2, u)}) = 0 \\ 所以f(S, T) = f(\begin{Bmatrix} s \end{Bmatrix}, V) = |f| $$

定理二：对于任意的可行流f，任意割[S, T]，|f| <= c(S, T)，即 最大流 <= 最小割

由上一个定理显然。

定理三：最大流最小割定理：

1.流f为最大流；

2.流f的残留网络中不存在增广路；

3.存在某个割[S, T]，|f| = c(S, T)。

以上三个条件互为充分必要条件。

证明：

$$ 通过证明 1 \Rightarrow 2 \Rightarrow 3 \Rightarrow 1即可证明原命题 $$

$1 \Rightarrow 2 : $

$$ 反证法：f为最大流，同时流f的残留网络存在增广路，那么在残留网络中一定存在一条从s到t的路径，路径上的边权大于0 \\ 那么就会存在一条新的可行流f’，|f+f’| > f，与f为最大可行流矛盾 $$

$ 2 \Rightarrow 3 : $

$$ 构造一个割：S为从s出发沿着大于0的边所能够走到的点的集合，T = V - S \\ 1.对于x \in S, y \in T, 一定有f(x, y) = c(x, y)，否则不满足构造条件 \\ 对于x \in T, y \in S，一定有f(x, y) = 0，否则就会出现一条从y到x的不为0的反向边，不满足构造条件 \\ 此时\sum{f(x, y)} = \sum{c(x, y)}(x \in S, y \in T)，即|f|=f(S,T) = c(S, T) $$

$ 3 \Rightarrow 1 : $

$$ |f| \leq 最大流，最大流 \leq c(S, T) = |f|，所以|f| = 最大流 $$

EK算法

复杂度：$ O(nm^2) $（伪复杂度，一般能够解决1000到10000的量级）

思路：

把原图直接当成残留网络来维护，每次bfs找一条增广路，并维护路径上边的最小值，沿着增广路找到一条路径，其路径上的最小边即为新的可行流，然后沿着路径处理边以及反向边即可。

代码：

#include<bits/stdc++.h>
using namespace std;
const int inf = 0x3f3f3f3f;
int n, m, S, T;
int idx = -1, fir[1010], to[20010], cap[20010], ne[20010];
int q[1010], d[1010], pre[1010];
bool book[1010];

void add(int u, int v, int c) {
    to[++ idx] = v, cap[idx] = c, ne[idx] = fir[u], fir[u] = idx;
    to[++ idx] = u, cap[idx] = 0, ne[idx] = fir[v], fir[v] = idx;
}

bool bfs()
{
    int qh = 0, qt = 1;
    memset(book, 0, sizeof book);
    d[S] = inf, book[S] = 1, q[0] = S;
    while(qh < qt)
    {
        int u = q[qh ++];
        for(int i = fir[u]; ~i; i = ne[i]) {
            int v = to[i];
            if(!book[v] && cap[i]) {
                d[v] = min(d[u], cap[i]);
                book[v] = 1;
                pre[v] = i;
                if(v == T) return true;
                q[qt ++] = v;
            } 
        }
    }
    return false;
}

int EK()
{
    int res = 0;
    while(bfs()) {
        res += d[T];
        for(int i = T; i != S; i = to[pre[i] ^ 1])
            cap[pre[i]] -= d[T], cap[pre[i] ^ 1] += d[T];
    }
    return res;
}

int main()
{
    memset(fir, -1, sizeof fir);
    scanf("%d%d%d%d", &n, &m, &S, &T);
    for(int i = 1,a,b,c; i <= m; i ++) {
        scanf("%d%d%d", &a, &b, &c);
        add(a, b, c);
    }
    printf("%d\n", EK());
    return 0;
}

Dinic算法

复杂度：$ O(n^2m) $（伪复杂度，一般能够解决10000到100000的量级）

思路：

相比于EK算法，dinic算法主要的优化地方在于它一次性可以找到多条增广路，一次性增广多条路，可以很大程度上优化。该算法主要用到两个优化思路，一个是分层图，将图进行分层，用于找到多条增广路。另外一个是当前弧优化，可以在dfs的时候免于搜索太多的无用边。

代码：

#include<bits/stdc++.h>
using namespace std;
const int inf = 0x3f3f3f3f;
int n, m, S, T;
int idx = -1, fir[10010], to[200010], cap[200010], ne[200010];
int q[10010], d[10010], cur[100010];

//d&#x6570;&#x7EC4;&#xFF0C;&#x7528;&#x4E8E;&#x5206;&#x5C42;&#x56FE;&#xFF0C;d[i]&#x4EE3;&#x8868;i&#x8282;&#x70B9;&#x5728;&#x5206;&#x5C42;&#x56FE;&#x7684;&#x7B2C;&#x51E0;&#x5C42;
//cur&#x6570;&#x7EC4;&#xFF0C;&#x5F53;&#x524D;&#x5F27;&#x4F18;&#x5316;&#xFF0C;cur[i]&#x4EE3;&#x8868;i&#x8282;&#x70B9;dfs&#x662F;&#x4ECE;cur[i]&#x5F00;&#x59CB;&#x904D;&#x5386;&#x8FB9;

void add(int u, int v, int c) {
    to[++ idx] = v, cap[idx] = c, ne[idx] = fir[u], fir[u] = idx;
    to[++ idx] = u, cap[idx] = 0, ne[idx] = fir[v], fir[v] = idx;
}

bool bfs()
{
    int qh = 0, qt = 1;
    memset(d, -1, sizeof d);
    q[0] = S, d[S] = 0, cur[S] = fir[S];
    while(qh < qt)
    {
        int u = q[qh ++];
        for(int i = fir[u]; ~i; i = ne[i]) {
            int v = to[i];
            if(d[v] == -1 && cap[i]) {
                d[v] = d[u] + 1;  //&#x8BB0;&#x5F55;&#x5206;&#x5C42;&#x56FE;
                cur[v] = fir[v];  //&#x8BB0;&#x5F55;&#x5F53;&#x524D;&#x5F27;
                if(v == T) return true;
                q[qt ++] = v;
            }
        }
    }
    return false;
}

int find(int u, int limit)
{
    if(u == T) return limit;
    int flow = 0;
    for(int i = cur[u]; ~i && flow < limit; i = ne[i]) {
        cur[u] = i;  //&#x4F18;&#x5316;&#x5F53;&#x524D;&#x5F27;
        int v = to[i];
        if(d[v] == d[u] + 1 && cap[i]) {
            int t = find(v, min(cap[i], limit - flow));
            if(!t) d[v] = -1;  //&#x5F53;&#x524D;&#x8FB9;&#x5DF2;&#x7ECF;&#x65E0;&#x6CD5;&#x8FC7;&#x6D41;&#x91CF;&#x4E86;&#xFF0C;&#x8E22;&#x51FA;&#x5206;&#x5C42;&#x56FE;
            cap[i] -= t, cap[i ^ 1] += t, flow += t;
        }
    }
    return flow;
}

int dinic()
{
    int res = 0, flow = 0;
    while(bfs()) while(flow = find(S, inf)) res += flow;
    return res;
}

int main()
{
    memset(fir, -1, sizeof fir);
    scanf("%d%d%d%d", &n, &m, &S, &T);
    for(int i = 1,a,b,c; i <= m; i ++) {
        scanf("%d%d%d", &a, &b, &c);
        add(a, b, c);
    }
    printf("%d\n", dinic());
    return 0;
}

最大流进阶

无源汇上下界可行流

问题：

给定一个包含 n 个点 m 条边的有向图，每条边都有一个流量下界和流量上界。

求一种可行方案使得在所有点满足流量平衡条件的前提下，所有边满足流量限制。

思路：

任何网络流的问题，都需要想办法转化成最基础的最大流问题，也就是说，如何建图是关键。

规定u(u, v)和l(u, v)表示u到v这条边的上界和下界。

根据题目：$ l(u, v) \leq f(u, v) \leq u(u, v) $

直观上，想办法转化成只有上界：$ 0 \leq f(u, v) - l(u, v) \leq u(u, v) - l(u, v) $

那么假设存在一种可行流，只要把每一条边的流量减去l(u, v)即可转化为最大流问题，现在只需要证明改变流量后的图的流是否为可行流。可行流满足两个条件：容量限制和流量守恒。根据不等式可以知道，容量限制是满足的，现在只需要证明对于每个点来说，满足流量守恒即可。

$ 那么对于任意一个点，流入该点的流量为 \sum{f(x, v)} - \sum{l(x, v)}，流出该点的流量为 \sum{f(v, x)} - \sum{l(v, x)} $

$ 对于原图明显 \sum{f(x, v)} = \sum{f(v, x)}，但是 \sum{l(x, v)} \neq \sum{l(v, x)} $

$ 所以需要想一个办法使得流量守恒，为了实现这个目标，创建两个虚拟源汇点S，T $

$ 令c_{in} = \sum{l(x, v)}为流入该点的边的下界，c_{out} = \sum{l(v, x)}为流出该点的边的下界 $

$ 分两种情况： $

$ 1.c_{in} > c_{out}，那么对于新图来说，流量减去下界之后，流入该点的流量变少了，所以需要虚拟源点S流c_{in} - c_{out}的流量流向该点 $

$ 2.c_{in} < c_{out}，类似的，需要该点流c_{out} - c_{in}的流量流向虚拟汇点T。 $

$ 也就是说：从S向每个1情况的点建一条容量为c_{in} - c_{out}的边，从每个2情况的点向T建一条容量为c_{out} - c_{in}的边 $

$ 为了满足流量守恒的情况，从S到T求出最大流后，必须从S流出的流量占满从S出去的边，流入T的边的流量也占满才能满足 $

$ 即\sum{f(S, x)} = \sum{c(S, x)}，\sum{f(x, T)} = \sum{c(x, T)} $

$ 现在证明\sum{c(S, x)} = \sum{c(x, T)}，这样我们只需要满足从S出去的流量满流即可 $

$ 对所有点有，c_{in1} - c_{out1}, c_{in2} - c_{out2}, …… , c_{inn} - c_{outn}，分成一情况和二情况 $

$ 左式（一情况）等于\sum{c_{in1}} - \sum{c_{out1}}，右式（二情况）等于\sum{c_{in2}} - \sum{c_{out2}} $

$ 这时只需要证明\sum{c_{in1}} - \sum{c_{out1}} = -(\sum{c_{in2}} - \sum{c_{out2}})（加负号是因为二情况为负数，需要取反证明相等） $

$ 移项\sum{c_{in1}} + \sum{c_{in2}} = \sum{c_{out1}} + \sum{c_{out2}}即\sum{c_{in}} = \sum{c_{out}} $

$ 考虑原图每一条边，每一条边都从一个点出发，到另外一个点，所以该边的下界对\sum{c_{in}}和\sum{c_{out}}，最后使得贡献相同 $

$ 原式得证 $

综合以上讨论，显然做法可行。

        scanf("%d%d%d%d", &a, &b, &l, &u);
        add(a, b, u - l);
        low[idx - 1] = l;
        fout[a] += l, fin[b] += l;
    }
    S = 0, T = n + 1;
    int tot = 0;  //tot&#x8BB0;&#x5F55;&#x6EE1;&#x6D41;&#x7684;&#x6D41;&#x91CF;
    for(int i = 1; i <= n; i ++)
        if(fin[i] > fout[i]) add(S, i, fin[i] - fout[i]), tot += fin[i] - fout[i];
        else if(fin[i] < fout[i]) add(i, T, fout[i] - fin[i]);
    int t = dinic();
    if(t < tot) return puts("NO")&0;  //&#x82E5;&#x6D41;&#x91CF;&#x4E0D;&#x6EE1;&#xFF0C;&#x65E0;&#x89E3;
    puts("YES");
    for(int i = 0; i < m << 1; i += 2) printf("%d\n", cap[i ^ 1] + low[i]);
    return 0;
}

有源汇上下界最大流

问题：

给定一个包含 n 个点 m 条边的有向图，每条边都有一个流量下界和流量上界。

给定源点 S 和汇点 T，求源点到汇点的最大流。

思路：

可以先想办法转化成无源汇，可以从T到S建一条容量无穷大的边，即可变成无源汇（这个称为循环流）

但是目前的问题是，无源汇的解法只能找到可行流$ f $，并不能找到最大流$ f_{max} $

先规定s和t为原图的源汇点，S和T为新图的源汇点。

一种比较直观的想法是先求S到T的满流，然后再去掉所有与S和T相连的边，然后在当前可行流的残留网络中求s到t的最大流，然后加上原本的可行流即可求出答案。

这种思路是没有问题的，问题在于逻辑上有一点点小错误。

首先要明白，要向得出当前可行流$ f $为最大流，必须要当前可行流$ f $的残留网络$ G_{f} $不存在增广路，这个思路的问题主要在于当前可行流的残留网络是新图的残留网络，而我们要找的是旧图的最大流，也就是旧图的残留网络不存在增广路才可以，而不是新图的残留网络不存在增广路。在新图求出满流之后我们只能得到新图的残留网络而不是旧图的残留网络。

这就在逻辑上有一点小错误，当然是可以证明在新图中从s到t的可行流是与原图从s到t的可行流是一一对应的。

$ 下面进行不严格证明： $

$ 设新图中的满流为f，对应旧图中的可行流为f_{old}，新图中s到t的可行流为f’，在新图中有0 \leq f + f’ \leq up - low $

$ 有low \leq f + low + f’ \leq up \Rightarrow low \leq f_{old} + f’ \leq up $

$ 相反也有类似的转化 $

$ 显然新图中从s到t的可行流都可以加到满流上，一直加到没有增广路为止即为最大流 $

代码：

#include<bits/stdc++.h>
using namespace std;
const int inf = 0x3f3f3f3f;
int n, m, S, T, s, t, fin[210], fout[210];
int idx = -1, fir[210], to[25000], cap[25000], ne[25000];
int q[210], d[210], cur[210];

void add(int u, int v, int c) {
    to[++ idx] = v, cap[idx] = c, ne[idx] = fir[u], fir[u] = idx;
    to[++ idx] = u, cap[idx] = 0, ne[idx] = fir[v], fir[v] = idx;
}

bool bfs()
{
    int qh = 0, qt = 1;
    memset(d, -1, sizeof d);
    d[S] = 0, q[0] = S, cur[S] = fir[S];
    while(qh < qt)
    {
        int u = q[qh ++];
        for(int i = fir[u]; ~i; i = ne[i]) {
            int v = to[i];
            if(d[v] == -1 && cap[i]) {
                d[v] = d[u] + 1;
                cur[v] = fir[v];
                if(v == T) return true;
                q[qt ++] = v;
            }
        }
    }
    return false;
}

int find(int u, int limit)
{
    if(u == T) return limit;
    int flow = 0;
    for(int i = cur[u]; ~i && flow < limit; i = ne[i]) {
        cur[u] = i;
        int v = to[i];
        if(d[v] == d[u] + 1 && cap[i]) {
            int temp = find(v, min(cap[i], limit - flow));
            if(!temp) d[v] = -1;
            cap[i] -= temp, cap[i ^ 1] += temp, flow += temp;
        }
    }
    return flow;
}

int dinic()
{
    int res = 0, flow;
    while(bfs()) while(flow = find(S, inf)) res += flow;
    return res;
}

int main()
{
    memset(fir, -1, sizeof fir);
    scanf("%d%d%d%d", &n, &m, &s, &t);
    for(int i = 1,a,b,u,l; i <= m; i ++) {
        scanf("%d%d%d%d", &a, &b, &l, &u);
        add(a, b, u - l);
        fin[b] += l; fout[a] += l;
    }
    S = 0, T = n + 1;
    int tot = 0;
    for(int i = 1; i <= n; i ++)
        if(fin[i] > fout[i]) add(S, i, fin[i] - fout[i]), tot += fin[i] - fout[i];
        else if(fin[i] < fout[i]) add(i, T, fout[i] - fin[i]);
    add(t, s, inf);
    int temp = dinic();
    if(temp < tot) return puts("No Solution")&0;
    temp = cap[idx];  //&#x6539;&#x53D8;&#x4E3A;&#x4ECE;t&#x5230;s&#x7684;&#x53CD;&#x5411;&#x8FB9;&#xFF0C;&#x5176;&#x5BB9;&#x91CF;&#x4E3A;&#x5FAA;&#x73AF;&#x6D41;&#x7684;&#x5BB9;&#x91CF;
    cap[idx] = cap[idx - 1] = 0;  //&#x5C06;t&#x5230;s&#x7684;&#x8FB9;&#x53BB;&#x6389;
    S = s, T = t;
    printf("%d\n", temp + dinic());
    //&#x8FD9;&#x91CC;&#x4E0D;&#x9700;&#x8981;&#x628A;&#x8FDE;&#x7740;S&#x548C;T&#x7684;&#x8FB9;&#x5220;&#x9664;&#xFF0C;&#x56E0;&#x4E3A;&#x5DF2;&#x7ECF;&#x6EE1;&#x8DB3;&#x6EE1;&#x6D41;&#x4E86;&#xFF0C;&#x6240;&#x4EE5;&#x589E;&#x5E7F;&#x4E0D;&#x4F1A;&#x589E;&#x5E7F;&#x5230;S&#x548C;T
    return 0;
}

有源汇上下界最小流

问题：

给定一个包含 n 个点 m 条边的有向图，每条边都有一个流量下界和流量上界。

给定源点 S 和汇点 T，求源点到汇点的最小流。

思路：

和最大流思路类似，做一些小变换即可。

求出满流之后，再求从t到s的最大流，然后减去即可。

直观上从t到s的最大流可以理解为最大能够退回的流量，即最小流。

using namespace std;
const int inf = 2147483647;
int n, m, S, T, s, t, fin[50010], fout[50010];

}

{

                d[v] = d[u] + 1;
                cur[v] = fir[v];
                if(v == T) return true;
                q[qt ++] = v;
            }
        }
    }
    return false;
}

int find(int u, int limit)
{
    if(u == T) return limit;
    int flow = 0;
    for(int i = cur[u]; ~i && flow < limit; i = ne[i]) {

        }
    }
    return flow;

多源汇最大流

问题：

给定一个包含 n 个点 m 条边的有向图，并给定每条边的容量，边的容量非负。

其中有 Sc 个源点，Tc 个汇点。

图中可能存在重边和自环。

求整个网络的最大流。

思路：

建立虚拟源汇点S和T，S到所有源点建立一条无穷大的边，所有汇点到T建立一条无穷大的边。

二分图匹配

思路：

对于题目给出的所有关系，左部点向右部点连一条权为1的边，然后建立虚拟源汇点S和T，S到每个左部点连一条权为1的边，每个右部点到T连一条权为1的边，这时最大流即为最大匹配。

匈牙利算法每一次只能找一条增广路，相当于EK，这时候用dinic算法会快很多。
