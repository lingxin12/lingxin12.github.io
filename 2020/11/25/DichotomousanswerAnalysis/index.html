贪心

简介：一种不断选择“局部最优的”的设计思想。

需要思考的问题：

1.贪心在什么情况下是正确的？（局部最优不一定等于全局最优）

2.全局最优解包含局部最优解（最优子结构性质）。

3.按某种决策方式不会丢失最优解（贪心选择性质）。

例题：

一、

问题：给出n个物体，第i个物体的重量为wi，选择尽量多的物体，使得总重量不超过C。

分析：很明显，选重的不如选轻的。

二、

问题：有n个物体，第i个物体重量为wi，价值为vi。在重量不超过C的情况下让总价值尽量的高。每个物体只取走一部分，重量与价值按比例折算。

分析：咋看上去01背包，实际上最后一句话可以贪心。因为一个物体是是可以无限割开，只拿1/3，那么价值也是1/3，这是只需要优先拿价值与重量之比大的即可。

三、

问题：有n个人，第i个人的重量为wi。每艘船的最大载重均为C，且最多只能乘坐两个人。用最少的船装所有人。

分析：直观上来看，人的总重量接近最大载重就很稳。那么很容易想到，最大重量和最小重量一起即可，但是最大重量可能太重。所以只需要考虑最小重量的人，优先去找能和他一起乘船的最大重量的人，更大重量的就不用考虑了，因为最小的都无法与其乘坐，那么其他的就更不用说了，让他们自己坐一条船即可。

六、

问题：数轴上有n个闭区间[ai,bi]，选择尽量少的区间覆盖指定线段[s,t]。

分析：通过以上几题的思考及训练，其实很容易分析了。

首先[s,t]左边肯定会被覆盖到，那么从左至右第一条覆盖的区间，a肯定小于等于s，那么我们只需要考虑a小于等于s的区间，尽量选更大的b即可。

二分查找（复习）

while(l<r)
{
    int mid = l+(r-l)/2;
    if(a[mid]>=x)
        r = mid;
    else
        l = mid + 1;
}

while(l<r)
{
    int mid = r-(r-l)/2;
    if(a[mid]<=x)
        l = mid;
    else
        r = mid - 1;
}

第一段代码是找第一个大于等于x的数，第二段代码是找最后一个大于等于x的数。

先看一下一个明显的优化，mid=l+(r-l)/2。

这里的原因很简单，因为C语言的除法是通过舍弃小数部分来得到整数的，如果商是正数是向下取整，如果商是负数是向上取整，为了避免没必要的麻烦，以后取中间值可以尽量都是用这种方法，保证都是向下取整。而且还有一个重要的功能就是防止溢出。

可以看到，第一段的和第二段的mid算法不一样，如果第二段的mid像第一段那样写的话，会死循环。

前缀和与差分

对一个序列a，从下标1开始，默认a[0]=0。

前缀和：s[i]=s[i-1]+a[i]。

差分：c[i]=a[i]-a[i-1]。

前缀和与差分为互逆运算，前缀和取差分得原数组，差分取前缀和得原数组。

二分答案

简介：

1.可用来解决一类最优化问题（比如：最小值最大之类的）

2.答案要具有“单调性”。

若最优解为S，那么对于任意x>S找不到对应方案满足，对于x<=S都能找到方案满足。

3.最最重要的是，可以把直接求解问题，化为判定该解是否有方案满足。

4.先猜一个解mid，若mid能实现，必有S>=mid，若不能，则S<mid。

例题：

一、跳石头（洛谷P2678，NOIP2015提高组）

分析：解析观察题目，我们很难直接去求出它的解。但是我们似乎很容易去验证一个解是否正确。

1.首先答案如果为x。

那么从左往右开始找石头，如果找到一段距离为y<x，那么必须把这段距离干掉，因为x为最短距离，那么我们就不能让y存在，那么此时到底移走前一个石头好呢还是后一个好呢，这时候就要使用贪心的思想了。

2.假设x为3，y为2，前面有若干个石头满足条件，接下来考虑这两块石头：

（左石头）·············（距离=2）·············（右石头）

如果移走左边，其实对已经满足条件的前边毫无影响。

但是这种情况：

（左石头）··············（距离=2）··········（右石头）··············（距离=2）··········· （下一个石头）

显然移走左边，下一段又得移右边，那还不如移走右石头，反正也不会影响左边的情况，这时候显然不会丢失最优解。

3.注意最后一段距离，不能移走端点石头。

4.如果我们发现答案如果为x时，移走的石头不超过M即答案满足，反之不满足。

5.如果答案为x满足，显然对于一个更小的答案也满足，因为石头按照x那样移走的话都可以，那么小于x的移走的石头肯定没x那时候多。如果答案为x不满足，那么答案肯定更小，这就是二分查找法。（这就是单调性啊，x越大，要移掉的石头越多）

代码：

#include<bits/stdc++.h>
#define enl puts("")
#define pii pair<int,int>
#define pdd pair<double , double >
#define pss pair<string , string >
#define pb push_back
typedef long long ll;
typedef long double ld;
typedef unsigned long long ull;
using namespace std;
const int maxn = 5e4+2;
 
int L,n,m,d[maxn];
 
bool check(int res)
{
    int front = 0, cnt = 0;
    for(int i=1;i<=n+1;i++) {
        if(d[i] - d[front] < res) {
            ++cnt;
            if(cnt > m) return false;
        }
        else front = i; 
    }
    return true;
}
 
int main()
{
    scanf("%d%d%d",&L,&n,&m);
    for(int i=1;i<=n;i++) scanf("%d",d+i);
    sort(d+1,d+n+1);
    d[n+1] = L; d[0] = 0;
    int l = 0, r = L;
    while(l<r) {
        int mid = (l+r>>1)+1;
        if(check(mid)) l = mid;
        else r = mid-1;
    }
    printf("%d\n",l);
    return 0;
}

三、聪明的质监员（洛谷P1314，2011NOIP提高组）

分析：

（注：yi的意思是区间中满足>=W的矿石数乘上这些矿石价值总和）

1.显然我们又可以发现，W似乎满足某种单调性（W越大，y越小），求使得|s-y|最小的最小W，显然用二分答案。

2.所以方法就很简单了，先选一个W，如果算出来的y值小于s，显然W可以更小，如果算出来的y值大于s，那么就尝试选小的W，看一下有没有使|s-y|更小的W值，这里要注意一下，如果y小于或者大于s，不能保证这个y不是离s最近的值，所以不能丢掉端点值，但是也要保证程序不死循环，不要重复验一个值。

3.检验y值就直接按照定义计算。

。。。。。。。。。。。。。。。

4.可惜这会超时，因为要计算m个区间，复杂度会达到O(logW(max)*k)，k为所有区间长度和，最坏会接近n*m。

5.计算y的时候，很容易发现，这是一个单纯的区间操作，显然可以用前缀和（没有修改没必要用线段树和树状数组），由于W是已知的，那么只需要扫一遍所有矿石，把价值wi>=W的做前缀和即可。

这时候复杂度仅为O(logW(max)*(n+m))。

代码：

#include<bits/stdc++.h>
#define enl puts("")
#define pii pair<int,int>
#define pdd pair<double , double >
#define pss pair<string , string >
#define pb push_back
typedef long long ll;
typedef long double ld;
typedef unsigned long long ull;
using namespace std;
const int maxn = 2e5+3;
int n,m,w[maxn],v[maxn],rl[maxn],rr[maxn],wa[maxn],va[maxn];
ll s;

ll check(int &mid)
{
    for(int i=1;i<=n;i++) if(w[i]>=mid) wa[i] = wa[i-1]+1; else wa[i] = wa[i-1];
    for(int i=1;i<=n;i++) if(w[i]>=mid) va[i] = va[i-1]+v[i]; else va[i] = va[i-1];
    ll ans = 0;
    for(int i=1;i<=m;i++) ans+=((ll)wa[rr[i]]-(ll)wa[rl[i]-1])*((ll)va[rr[i]]-(ll)va[rl[i]-1]);
    return ans;
}

int main()
{
    scanf("%d%d%lld",&n,&m,&s);
    for(int i=1;i<=n;i++) scanf("%d%d",w+i,v+i);
    for(int i=1;i<=m;i++) scanf("%d%d",rl+i,rr+i);
    int l = 1e7,r = 0;
    for(int i=1;i<=n;i++) l = min(l,w[i]), r = max(r,w[i]);
    r+=1;
    while(l < r-1)
    {
        int mid = l+r>>1;
        ll res = check(mid);
        if(res < s) r=mid;
        else if(res > s) l=mid;
        else {
            printf("%d\n",mid);
            return 0;
        } 
    }
    printf("%lld\n",abs(check(l)-s)<abs(check(r)-s)?abs(check(l)-s):abs(check(r)-s));
     return 0;
}

四、借教室（洛谷P1083，2012NOIP提高组）

分析：

1.看下来，有一点特别需要注意，那就是借教室的原则是先到先得，就是如果我先来并满足条件，那么后来的你如果不满足的话，那么你得修改订单。

2.那么这里显然满足一种单调性，对于前x个订单，x越小似乎越容易满足，x越多似乎越难满足。

3.如果前x个订单满足，那么x就可以更大，如果不能满足，那么x就得更小。

（其实这道题很bug，那就是如果有多个时段都不满足的话，那就答案就是最前面哪个，因为原则是按顺序处理的，这题需要找的是最需要修改订单的那个人，即最先不满足条件的那个人。如果答案为x时不能满足，我们是不能直接输出到底是谁不满足的，得要继续缩小，直到l==r，然后就知道到底到哪个订单就不满足了）

4.这里最主要的就是判断前x个订单究竟满不满足。

5.如果按照暴力的方法修改订单，显然会超时，但是注意到这里是多次区间修改，一次查询。

（这和上一题有异曲同工之处，那就是上一题是没有区间修改，但有多次查询）

显然用线段树和树状数组比较麻烦。

对于这种情况，有一种更好的解决办法，那就是差分数组。

a数组：1 2 -4 5 6 3

c数组：1 1 -6 9 1 -3

很显然，如果我们要修改区间[a,b]，比如加上x，我们只需要c[a]+=x，c[b]-=x即可，那么区间修改就可以简化为两点修改。（如果减去x，我们只需要c[a]-=x，c[b]+=x即可）

然后c数组做一次前缀和就可以得到a数组，最后扫描a数组做一次查询，看其是否大于等于di即可。

代码：

#include<bits/stdc++.h>
#define enl puts("")
#define pii pair<int,int>
#define pdd pair<double , double >
#define pss pair<string , string >
#define pb push_back
typedef long long ll;
typedef long double ld;
typedef unsigned long long ull;
using namespace std;
const int maxn = 1e6+3;
int n,m,r[maxn],d[maxn],s[maxn],t[maxn],c[maxn],a[maxn];

bool check(int res)
{
    for(int i=1;i<=n;i++) c[i] = r[i]-r[i-1];
    for(int i=1;i<=res-1;i++) c[s[i]]-=d[i], c[t[i]+1]+=d[i];
    for(int i=1;i<=n;i++) a[i] = a[i-1]+c[i];
    for(int i=1;i<=n;i++) if(a[i] < 0) return false;
    return true;
}

int main()
{
    scanf("%d%d",&n,&m);
    for(int i=1;i<=n;i++) scanf("%d",r+i);
    for(int i=1;i<=m;i++) scanf("%d%d%d",d+i,s+i,t+i);
    for(int i=1;i<=n;i++) c[i] = r[i]-r[i-1];
    for(int i=1;i<=m;i++) c[s[i]]-=d[i], c[t[i]+1]+=d[i];
    for(int i=1;i<=n;i++) a[i] = a[i-1]+c[i];
    int ok = 1;
    for(int i=1;i<=n;i++) if(a[i] < 0) {
        ok = 0;
        break;
    }
    if(ok) {
        printf("0\n");
        return 0;
    }
    int l = 1, r = m;
    while(l<r)
    {
        int mid = r-(r-l)/2;
        if(check(mid)) l = mid;
        else r = mid - 1;
    }

    printf("-1\n%d\n",l);
     return 0;
}
