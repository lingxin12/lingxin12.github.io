A题：

题意：

给一个n*m的图，每个方格涂上黑色或白色，一个格子的相邻格子有与其不同的颜色，则该格子的颜色为有效的，求一个涂色方法，使得有效黑格子正好比有效白格子多1

B题：

题意：

给只含有-1，0，1的元素的a数组，和一个b数组。

操作若干次，每次可以选择(i,j)，使得aj = ai + aj（其中1<=i<j<=n)。

问是否若干次操作后，把a变成b。

题解：

若第一个元素不同那就不可能了。

可以从后往前遍历，比较ai与bi的大小，看看前面是否有所需要的1，-1即可。

代码：

#include<bits/stdc++.h>
typedef long long ll;
using namespace std;
const int maxn = 1e5 + 3;
int T,n;
int a[maxn],b[maxn];
int book[2][maxn];
//book[0][i]&#x8BB0;&#x5F55;&#x5230;i&#x65F6;&#x4E00;&#x5171;&#x6709;&#x51E0;&#x4E2A;1
//book[1][i]&#x8BB0;&#x5F55;&#x5230;i&#x65F6;&#x4E00;&#x5171;&#x6709;&#x51E0;&#x4E2A;-1
int main()
{
    scanf("%d",&T);
    while(T--)
    {
        memset(book,0,sizeof(book));
        scanf("%d",&n);
        for(int i=1;i<=n;i++){
            scanf("%d",&a[i]);
            book[0][i] = book[0][i-1];   //&#x8FD9;&#x91CC;&#x6CE8;&#x610F;&#x627F;&#x63A5;&#x4E0A;&#x4E00;&#x4E2A;book&#x503C;
            book[1][i] = book[1][i-1];   //
            if(a[i] == 1) book[0][i]++;
            if(a[i] == -1) book[1][i]++;
        }
        for(int i=1;i<=n;i++) scanf("%d",&b[i]);
        if(a[1] != b[1]){
            printf("NO\n");
            continue;
        }
        int ok = 1;
        for(int i=n;i>=2;i--)
            if(a[i]>b[i]&&book[1][i-1]==0 || a[i]<b[i]&&book[0][i-1]==0){
                ok = 0;
                break;
            }
        if(ok) printf("YES\n");
        else printf("NO\n");
    }
    return 0;
}

C题：*****

题意：

给一个序列a，求其子段（包括自己）为好段的个数，一个好的序列即为所有元素之和不为0。

题解：

sum为0的问题，可以用map来记录最后一次sum值所在的位置。这样子循环到i时，如果sum值已经出现，那么就可以知道从上次出现的sum值的位置一直到i这一段不是好段，那么就可以直接i-last算出res。

last用来记录最后一个非好段的开头。

代码：

#include<bits/stdc++.h>
typedef long long ll;
using namespace std;
const int maxn = 2e5 + 3;
int n;
ll a[maxn];
map<ll,ll> mp;
int main()
{
    scanf("%d",&n);
    for(int i=1;i<=n;i++) scanf("%lld",&a[i]);
    mp[0] = 1; ll sum = 0,res = 0,las = 0;
    for(int i=1;i<=n;i++)
    {
        sum += a[i];
        if(mp[sum]) las = max(las,mp[sum]);
        res += i-las;
        mp[sum] = i+1;
    }
    printf("%lld\n",res);
    return 0;
}

D题：

题意：

n个人站成一排，每个人向左站或向右站，每一秒至少选取一对相对而站的人变成相背而站。问是否能正好k秒后每两个相邻的人相背而站。输出每一秒向左转的人的位置。若无解，输出-1。

n<=3000,k<=300,0000（解最多在n*n秒内）

题解：

当时看错题目了，没有看到至少两个字，害我以为有什么办法能够直接算出给定序列的k值，然后判断。

既然是至少，那么k就会在一个范围内，如果把每一秒中所有能够转向的人都转向就是最小的，把每一步转向都分开，那么就是最大的，k不在这个范围内就past掉。

（其实就是暴力解法，毕竟题目给了2s的时间，把所有步骤都存在一个vector中）
