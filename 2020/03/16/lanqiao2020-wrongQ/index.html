将小明的数分解后，可以表示成3^15*5*7^7，这样考虑i，j，k在30次以内枚举。

然后将数从大到小排序，但是出现错误。因为数大得无法存入。

然后考虑在15次进行枚举，并排序。

可是又出现了问题：有些数也很大无法存入，同时3^16也小于上数，故有误。

但是经过仔细思考后发现，其实只需要把小于小明的数记下来一共有几个即可，考虑到，小明的数远远小于long long的最大值，则可以对ans一边乘一边判断是否大于小明的数，如果大于就没有必要继续往下乘了。

代码如下：

#include<bits/stdc++.h>
using namespace std;
typedef long long ll;
int main()
{
    int n = 0;  ll num = 59084709587505;
    for(int i=0;i<=40;i++)
        for(int j=0;j<=40;j++)
            for(int k=0;k<=40;k++)
                {
                    ll ans = 1;  int ok = 1;
                    for(int a=1;a<=i;a++){
                        ans = ans * 3;
                        if(ans > num){
                            ok = 0;
                            break;
                        }
                    }
                    if(ok) for(int a=1;a<=j;a++){
                        ans = ans * 5;
                        if(ans > num){
                            ok = 0;
                            break;
                        }
                    }
                    if(ok) for(int a=1;a<=k;a++){
                        ans = ans * 7;
                        if(ans > num){
                            ok = 0;
                            break;
                        }
                    }
                    if(ok) ++n;
                }
    cout << n << endl;
    return 0;
}

最后注意输出答案时，要去除1这种情况，因为他不能分解成3，5，7的乘积。
